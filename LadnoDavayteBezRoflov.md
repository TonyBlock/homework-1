# Страница входа

Ссылка: https://brrrello.ru/login

  - Поле "Логин"
    - по клику на инпут текст "Логин" уезжает над инпут;
    - по клику на инпут заполняется полоска инпута;
    - поле автозаполняется сохраненным логином при его наличии;

    - Цветовая валидация инпута ок:
      - Длина: валидно с 3 по 20 включительно, остальное - невалидно;
      - Содержание: латиница и цифры валидны, кириллица и специальные символы невалидны;
    
    - По снятию фокуса с инпута при введенном логине текст "Логин" не возвращается в инпут;
    - По снятию фокуса с инпута при введенном логине полоска не сбрасывается в исходное состояние;

  - Поле "Пароль"
    - по клику на инпут текст "Пароль" уезжает над инпут;
    - по клику на инпут заполняется полоска инпута;
    - поле автозаполняется сохраненным паролем при его наличии;

    - Цветовая валидация инпута не ок:
      - **Баг** Длина: невалидно до 6 символов включительно, дальше валидно;
      - ***Ожидаемое поведение*** Длина: валидно с 6 до 25 включительно, остальное невалидно;
      - Содержание: пароль, состоящий из латиницы, валиден;
      - Содержание: пароль, начинающийся с цифры, невалиден;
      - Содержание: пароль, состоящий из латиницы с цифрами не на первой позиции, валиден;
      - Содержание: пароль, состоящий из кириллицы, невалиден;
    
    - По снятию фокуса с инпута при введенном пароле текст "Пароль" не возвращается в инпут;
    - По снятию фокуса с инпута при введенном пароле полоска не сбрасывается в исходное состояние;

  - Ссылка "Создать аккаунт" ок:
    - корректно ведет на https://brrrello.ru/register;

  - По клику на кнопку "Войти" с корректной парой логин/пароль происходит редирект на https://brrrello.ru/boards;

  - По клику на кнопку "Войти" с некорректной парой логин/пароль редиректа не происходит:
    - На странице показываются два сообщения об ошибке с текстом "Неверный логин или пароль";

  - По клику на кнопку "Войти" с незаполненными инпутами показываются два сообщения об ошибке:
    - Под инпутом "Логин": "Введите логин длиной от 3 до 20 символов";
    - Под инпутом "Пароль": "Введите пароль длиной от 6 до 25 символов";

  - По клику на кнопку "Войти" с незаполненным инпутом "Логин" показывается одно сообщение об ошибке:
    - Под инпутом "Логин": "Введите логин длиной от 3 до 20 символов";
    - Введенное пользователем значение инпута "Пароль" сохраняется;

  - По клику на кнопку "Войти" с незаполненным инпутом "Пароль" показывается одно сообщение об ошибке:
    - Под инпутом "Пароль": "Введите пароль длиной от 6 до 25 символов";
    - Введенное пользователем значение инпута "Логин" сохраняется;

# Страница со списком команд

Ссылка: https://brrrello.ru/boards

## Элементы страницы

  - "Личное пространство" не ок: 
    - Всегда отображается первой в списке команд;
    - У пространства отсутствует кнопка "Удалить";
    - У пространства отсутствует кнопка "Редактировать";
    - У пространства отсутствует счетчик участников и кнопка "Добавить участника";
    - У пространства отображается кнопка добавления доски:
      - Кнопка добавления доски ведет на модальное окно создания доски со значением "Личное пространство" в поле "Команда";
    - В пустом личном пространстве отображается текст "Похоже досок пока нет. Создайте Ваш первый проект в личном пространстве.";
      - Ссылка "Создайте" ведет на модальное окно создания доски со значением "Личное пространство" в поле "Команда"
    - **Баг** При смене логина пользователя не меняется название личного пространства;
    - ***Ожидаемое поведение*** При смене логина пользователя название личного пространства обновляется с учетом нового логина;

  - Команда ок:
    - Старые команды отображаются в алфавитном порядке;
    - Новые команды добавляются снизу и при перезагрузке перерисовываются в алфавитном порядке;
    - В пустой команде отображается текст "Похоже досок пока нет. Создайте первый проект команды "..."":
      - Ссылка "Создайте" ведет на модальное окно создания доски со значением названия текущей команды в поле "Команда";
    - У команды отображается кнопка "Удалить":
      - При клике на кнопку отображается модальное окно удаления текущей команды;
    - У команды отображается кнопка "Редактировать":
      - При клике на кнопку отображается модальное окно редактирования команды со значением названия текущей команды в поле "Название команды";
    - У команды отображается кнопка добавления доски:
      - Кнопка добавления доски ведет на модальное окно создания доски со значением названия текущей команды в поле "Команда";
    - У команды отображается кнопка "Добавить участника" и счетчик участников команды:
      - При клике на кнопку отображается модальное окно добавления участника в текущую команду;

  - "Остальные доски" ок:
    - Отображается только в том случае, если есть доски, в которые приглашен пользователь и при этом пользователь не является участником команды, создавшей доску;
    - Отображаются последней в списке команд;
    - У раздела отсутствует кнопка "Удалить";
    - У раздела отсутствует кнопка "Редактировать";
    - У раздела отсутствует счетчик участников и кнопка "Добавить участника";
    - При уходе пользователя из доски доска исчезает из раздела;

Вся страница:

  - Кнопка доски ведет на доску с айди, совпадающим с отображающимся на списке досок;
  - Кнопка "Создать команду" открывает модальное окно создания команды;

## Модальное окно создания команды

 - Модальное окно создания команды ок:
  - По клику на крестик модальное окно закрывается, не создавая команду;
  - По клику на кнопку "Создать" с пустым инпутом "Название команды" отображается ошибка с текстом "Название команды слишком короткое";
  - По клику на кнопку "Создать" с любым текстом в инпуте "Название команды" модальное окно закрывается, создавая команду с введенным названием;

## Модальное окно удаления команды

 - Модальное окно удаления команды ок:
  - По клику на крестик модальное окно закрывается, не удаляя команду;
  - По клику на кнопку "Нет, отменить" модальное окно закрывается, не удаляя команду;
  - По клику на кнопку "Да, удалить" модальное окно закрывается, удаляя команду;

## Модальное окно редактирования команды

 - Модальное окно редактирования команды не ок:
    - **Баг** Отображается текст "Создание команды";
    - ***Ожидаемое поведение*** Отображается текст "Редактирование команды";
  - По клику на крестик модальное окно закрывается, не изменяя команду;
  - По клику на кнопку "Сохранить" с пустым инпутом "Название команды" отображается ошибка с текстом "Название команды слишком короткое";
  - По клику на кнопку "Сохранить" с любым текстом в инпуте "Название команды" модальное окно закрывается, изменяя название команды на введенное;

## Модальное окно добавления участника в команду

 - Модальное окно добавления участника в команду ок:
  - В списке участников отображаются все участники команды;
  - По клику на участника он исключается из команды;
  - По вводу трех символов в инпут происходит поиск участника. По добавлению символов поисковые результаты корректируются;
  - По клику на участника на поиске иконка плюса заменяется на иконку галочки и участник добавляется в команду;
  - По клику на крестик модальное окно закрывается, не изменяя список участников;

## Модальное окно создания доски 

 - Модальное окно создания доски ок:
  - По клику на крестик модальное окно закрывается, не создавая доску;
  - По клику на кнопку "Создать" с пустым инпутом "Название доски" отображается ошибка с текстом "Название доски слишком короткое";
  - В выпадающем меню "Команда" отображается список команд пользователя;
  - По клику на кнопку "Создать" с любым текстом в инпуте "Название доски" модальное окно закрывается, создавая доску в соответствующей команде;

## Модальное окно добавления пользователя в доску

  - Поле для ввода имени пользователя oк:
    - Изначально пустое;
    - Если вводимое имя начинается с латинской буквы, то ничего не появляется;
    - Если вводимое имя начинается с не-латинской буквы или цифры, то появляется надпись "Введите логин, содержащий только латинские буквы";
    - При количестве символов в поле от 3 до 20 включительно список ниже поля содержит пользователей, чей логин начинается с введенных символов;
    - При количестве символов в поле от 0 до 2 включительно список ниже поля содержит список участников, добавленных конкретно на доску;
    - При количестве символов в поле свыше 20, то появляется надпись "Введите логин длиной от 3 до 20 символов";
  - Список участников, добавленных конкретно на доску или список найденных пользователей
    - Элемент списка участников/найденных пользователей;
      - слева содержит аватар пользователя;
      - правее от аватара располагается его логин;
      - в правой части элемента располагается плюс или галочка в зависимости от того, добавлен ли пользователь к доске;
      - по нажатию на плюс пользователь добавляется к доске;
      - по нажатию на галочку пользователь удаляется из доски;
  - Поле со ссылкой приглашением;
  - Кнопка копирования ссылки;
  - Кнопка генерации новой ссылки;

## Модальное окно создания колонки

  - Поле для ввода названия колонки
    - Изначально пустое;
    - Очищается после неудачной попытки создать колонку;
  - Кнопка "Создать" ок:
    - Если название колонки длиной от 1 до 40 символов включительно, то по нажатию на кнопку модальное окно закрывается, а на доске появляется новая колонка;
    - Если название колонки не введено, то при попытке создать колонку выведется сообщение "Название колонки слишком короткое";
    - Если название колонки свыше 20 символов в длину, то при попытке создать колонку выведется сообщение "Название колонки слишком длинное";

## Модальное окно настроек доски

  - Поле с названием доски
    - Изначально заполнено текущим названием доски;
  - Поле с описанием доски
    - Изначально заполнено текущим описанием доски;
  - Кнопка "Удалить доску"
    - При её нажатии она заменяется двумя кнопками "Удалить" и "Не удалять";
      - Если нажать "Удалить, то доска будет удалена и пользователя перебросит на главный экран;
      - Если нажать "Не удалять", то две кнопки заменятся обратно на одну;
  - Кнопку "Сохранить" ок:
    - Если длина названия доски от 1 до 60 символов, до изменения применятся и модальное окно закроется;
    - Если указано пустое название доски, то при попытке сохранить будет выведено "Название доски слишком короткое", а само сохранение будет отклонено;
    - Если длина названия доски свыше 60 символов, то при попытке сохранить будет выведено "Название доски превышает 60 символов", а само сохранение будет отклонено;

    - если название колонки не введено, то при попытке создать колонку выведется сообщение "Название колонки слишком короткое"
    - если название колонки свыше 20 символов в длину, то при попытке создать колонку выведется сообщение "Название колонки слишком длинное"

## Модальное окно создания карточки

* Нажатие ЛКМ на "крестик" ок:
  * Окно создания карточки закрывается, отображается текущая доска
* Нажатие ЛКМ за пределами модального окна ок:
  * Окно создания карточки закрывается, отображается текущая доска;
* Кнопка "Создать" не ок:
  * Никакие данные в инпуты с названимем не вводились:
    * Cообщение об ошибке "Название карточки слишком короткое";
  * В инпут "Название карточки" введен слишком длинный текст (100 символов):
    * Сообщение об ошибке "Название карточки слишком длинное"
  * В инпут "Название карточки" введен допустимый, длинный текст (25 символов):
    * **Баг:** карточка успешно создана, длинное название скрылось за троеточием на карточке, но в модальном окне текст не скрыт за троеточием и уходит за скроллзону. 
    * **Ожидаемый результат:** успешное создание карточки, длинное название скрыто за троеточием на карточке и в модальном окне карточки.
    * **Приложение**: https://ibb.co/vjgMqrW
  * В инпут "Название карточки" введен текст не допустимой длины (100 символов), в инпут описание введен допустимый текст (300 символов) и выбрана сегодняшняя дата делайна:
    * **Баг:** название карточки, делайн и описание были сброшены.
    * **Ожидаемый результат:** сообщение об ошибке "Название карточки слишком короткое", текст описания, название карточки и дата дедлайна остались в инпуте.
  * Ввод в инпуты формы корректных данных (название, описание, дедлайн):
    * Карточка успешно создается, на карточке отображается значок описания и дедлайна.
  * Ввод в инпут формы корректных данных (название, описание):
    * Карточка успешно создается, на карточке отображается значок описания.
  * Ввод в инпут формы корректных данных (название, дедлайн):
    * Карточка успешно создается, на карточке отображается значок дедлайна.

* Инпут ввода дедлайна не ок:
  * Ввод в инпут не корректных данных - *30.02.2022 10:10*
    * **Баг:** дата не исправлена
    * **Ожидаемый результат:** дата будет исправлена на ближайшую корректную
  * Ввод в инпут не корректный данных - *30.55.2011 10:80*:
    * дата будет исправлена на ближайшую корректную, и отобразится в созданной карточке.
  * Ввод в ипут корретных данных
    * Ничего не происходит, введенные значения отображаются в формате: *DD.MM.YYYY HH:MM*.


## Модальное окно карточки

> Принципы тестирования редактирования карточки в модальном окне карточки совершенно такие же как и в модальном окне создания карточки. Замеченные баги при создании карточки также верны и для редактирования в модальном окне карточки.

* Нажатие ЛКМ на "крестик" ок:
  * Окно создания карточки закрывается, отображается текущая доска
* Нажатие ЛКМ за пределами модального окна ок:
  * Окно создания карточки закрывается, отображается текущая доска;
* Нажатие ЛКМ на "крестик" ок:
  * Окно карточки закрывается, отображается текущая доска
* Нажатие ЛКМ за пределами модального окна ок:
  * Окно карточки закрывается, отображается текущая доска.

* Кнопка "Участники" ок:
  * Нажатие открывает модальное окно "Добавить пользователя в карточку"

* Кнопка добавления тега на карточку ок:
  * Нажатие приводит к отображению поверх модального окна карточки модального окно с тегами.
* Значок ранее добавленного тега ок:
  * Нажатие на значок приводит к тому, что поверх модального окна карточки отображается модальное окно со тегов.

* Вложения не ок:
  * Загрузка вложения не корректного размера (500 мб):
    * Выводится сообщение об ошибке "Слишком большой размер вложения"
  * Загрузка вложения размером 0 байт:
    * Вложение успешно загружается и отображается в списке "Вложения", название вложения такое же как и у загруженного пользователем файла
  * Загрузка вложения, в названии которого есть символы кириллицы:
    * Вложение успешно загружается и отображается в списке "Вложения", название вложения такое же как и у загруженного пользователем файла.
  * Нажатие на значок загрузки у вложения с названием содержащим символы кириллицы:
    * Загружается вложение, в названии кириллические символы заменены на соответствующие по звучанию латинские символы
  * Нажатие на значок удаления рядом с вложением:
    * Вложение удаляется из списка вложений
  * Загрузка вложений с одинаковыми названиями и последующая их загрузка пользователем:
    * Содержимое вложений не изменилось
  * Нажатие на значок удаления вложения когда в списке "Вложения" остался один элемент:
    * Вложение удалено, список больше не отображается
  * Загрузка вложения `.docx`:
    * **Баг:** вложение не загружено
    * **Ожидаемый результат:** вложение загружено и отображается в списке вложений

* Чеклисты не ок:
  * Нажатие на кнопку "Новый чек-лист":
    * Создается чеклист с названием по умолчанию в списке "Чек листы"; на карточке появляется значок чеклиста
  * Нажатие на значок редактирования чеклиста:
    * Название чеклиста заменяется на инпут с названием чеклиста, значок редактирования (карандаш) заменяется на значок сохранения (дискета).
  * Ввод нового названия чеклиста большой длины (1000 символов) и нажатие на значок "Сохранить" рядом с чеклистом:
    * **Баг:** чеклист успешно переименовывается
    * **Ожидаемый результат:** вывод сообщения о недопустимой длине названия
  * Удаление названия чеклиста и нажатие на значок "Сохранить" рядом с чеклистом:
    * **Баг:** чеклист успешно переименовывался, название отсутствует
    * **Ожидаемый результат:** вывод сообщения о недопустимой длине названия
    * **Приложение**: https://ibb.co/pwpZ4Pq
  * Ввод корректного названия чеклиста, длиной 25 символов и нажатие на значок "Сохранить" рядом с чеклистом:
    * Чеклист успешно переименован
  * Нажатие на значок удаления чеклиста:
    * Чеклист удален вместе со всеми элементами
  * Нажатие на значок "+" рядом с чеклистом:
    * Создается элемент чеклиста с названием по умолчанию
  * Нажатие на значок редактирования элемента чеклиста:
    * Название элемента чеклиста заменяется на инпут с названием чеклиста, значок редактирования (карандаш) заменяется на значок сохранения (дискета).
  * Ввод нового названия элемента чеклиста большой длины (1000 символов) и нажатие на значок "Сохранить" рядом с чеклистом:
    * **Баг:** элемент чеклиста успешно переименовывается
    * **Ожидаемый результат:** Вывод сообщения о недопустимой длине названия
  * Удаление названия элемента чеклиста и нажатие на значок "Сохранить" рядом с элементом чеклистом:
    * **Баг:** элемент чеклиста успешно переименовывался, название отсутствует
    * **Ожидаемый результат:** вывод сообщения о недопустимой длине названия
    * **Приложение**: https://ibb.co/pwpZ4Pq
  * Ввод корректного названия элемента чеклиста, длиной 25 символов и нажатие на значок "Сохранить" рядом с элементом чеклиста:
    * Элемент чеклиста чеклист успешно переименован
  * Нажатие на чек-бокс чеклиста:
    * Статус чекбокса переключился
  * Нажатие на значок удаления элемента чеклиста:
    * Элемент удалился

* Комментарии:
  * Нажатие на значок "Добавить", в инпут комментария не введен текст:
    * **Баг:** создается пустой комментарий
    * **Ожидаемый результат:** сообщение о не корректной длине комментария
  * Ввод комментария длиной 100 символов:
    * **Ожидаемый результат:** комментарий успешно создан и корректно отображается
    * **Баг:** комментарий успешно создан, но текст отображается в одну строчку и скрывается под скролл зоной
    * **Приложение**: https://ibb.co/RCWFzvz
  * Нажатие на значок редактирования комментария:
    * **Ожидаемый результат:** комментарий заменяется на инпут с текстом комментария, значок редактирования заменяется на значок сохранения
    * **Баг:** комментарий заменяется на инпут с текстом комментария, замена значка не происходит, вместо этого отображаются оба значка.
    * **Приложение**: https://ibb.co/4mxyQcR
  * Ввод комментария нулевой длины и нажатие на значок "Сохранить", при редактировании комментария:
    * **Ожидаемый результат:** вывод сообщения о не корректной длине комментария
    * **Баг:** сообщения не выводится, комментарий изменяется на указанный
  * Ввод комментария корректной длины и нажатие на значок "Сохранить", при редактировании комментария:

## Модальное окно "Редактирование тега"

* Нажатие ЛКМ на "крестик":
  * модальное окно "Редактирование тега" закрывается, отображается модальное окно карточки

* Нажатие ЛКМ за пределами модального окна "Редактирование тега":
  *  модальное окно "Редактирование тега" закрывается, отображается модальное окно "Теги"

* Удаление из инпута "Название" содержимого и нажатие кнопки "Сохранить":
  * выводится сообщение о недопустимой длине тега

* Введение в инпут "Название" строки более чем в 40 символов и нажатие кнопки "Сохранить":
  * выводится сообщение о недопустимой длине тега

* Нажатие на значок цвета отмеченного галочкой
  * **Баг:** "галочка" исчезает со значка цвета
  * **Ожидаемый результат:** не происходит ничего

* Нажатие на значок цвета не отмеченного галочкой
  * "галочка" исчезает с предыдущего значка цвета и появляется на нажатом значке

Цвет редактируемого тега и его название в окне "Теги" изменились, изменился цвет тега у всех карточек доски, ранее отмеченных данным тегом, изменился цвет соответствующего тега на странице доски также изменился цвет данного тега в модальном окне "Теги" доски.

* Нажатие на кнопку "Удалить":
  * Закрытие модального окна "Редактирование тега", отображается окно "Теги". Тег исчезает из списка тегов модального окна "Теги", из списка тегов в модальном окне карточки, а также из модального окна "Теги" отрывающегося по кнопке "Теги" на странице доске и всех карточек ранее отмеченных данных тегом.

  * Закрытие модального окна, отображается окно "Теги", в окне теги появляется новый тег с корректным цветом и названием

## Модальное окно "Добавить пользователя в карточку"

* Нажатие ЛКМ на "крестик":
  * модальное окно "Добавить пользователя в карточку" закрывается, отображается модальное окно карточки

* Нажатие ЛКМ за пределами модального окна "Добавить пользователя в карточку":
  * модальное окно "Добавить пользователя в карточку" закрывается, отображается модальное окно карточки

* Ввод в инпут поиска имени пользователя длиной менее 3 символа:
  * поиска не выполняется

* Ввод в инпут поиска части имени пользователя (более 3х символов) не состоящего в доске:
  * пользователь не будет найден

* Ввод в инпут поиска части имени пользователя (более 3х символов) состоящего в доске:
  * пользователи будут добавлены в списке под инпутом, при этом пользователи не назначенные как исполнители задачи отмечены значком "+", а назначенные значком "галочка".

* Нажатие на элемент в списке найденных пользователей (пользователь не назначен на карточку):
  * пользователь назначается как исполнитель задачи, его аватар отображается на карточке

* Нажатие на элемент в списке найденных пользователей (пользователь назначен на карточку):
  *  пользователь удаляется из списка исполнителей задачи: его аватар больше не отображается на карточке

* Ввод в инпут поиска символов кириллицы:
  * вывод сообщения об ошибке "Введите логин, содержащий только латинские буквы"

* Ввод инпут поиска текста длиной более 20 символов:
  * ывывод сообщения об ошибке "Введите логин длиной от 3 до 20 символов"

* Посимвольный ввод имени пользователя в инпут поиска:
  * список найденных пользователей обновляется

* Нажатие на значок копирования рядом ссылкой-приглашением:
  * ссылка пере генерации ссылки в буфер обмена

* Нажатие на значок обновления ссылки-приглашения:
  * ссылка в инпуте была изменен



# Общее

## Navbar

* Состояние навбара для не авторизованного пользователя:
  * Навбар содержит 2 пунтка: "Войти", "Зарегистрироваться".

* Состояние навбара для авторизованного пользователя
  * Навбар содержит 3 пункта: "Доски", блок "Аватар + имя пользователя", "Выход".

* Нажатие на значок brrrello для авторизованного пользователя:
  * Открывается список досок.

* Нажатие на значок brrrello для не авторизованного пользователя:
  * Открывается страница входа.

* Нажатие на кнопку "Войти":
  * Открывается страница входа.

* Нажатие на кнопку "Зарегистрироваться":
  * Открывается станица регистрации.

* Нажатие на кпоку "Доски" :
  * Открывается страница со списком досок.

* Нажатие на блок "имя пользователя + аватар":
  * Открывается страница профиля пользователя

* Состояние навбара на мобильных устройствах (ширина экрана менее 500px):
  * Элементы навигации скрыты в гамбургер-меню.


* Нажатие на кнопку-гамбургер на мобильных устройствах:
  * Приводит к раскрытию меню.

* При ручном уменьшение экрана браузера с десктопной версии (ширина экрана 500px+) до мобильной (500px и менее):
  * **Ожидаемый результат:** элементы навигации скрываются в гамбургер-меню, меню при этом скрыто.
  * **Баг:** элементы регистрации скрыты в меню, но само меню раскрыто. Баг проявляется *только* в браузере `Chromium 99.0.xxxx.xx`.
